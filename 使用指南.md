# 半自动剪辑体系4.8 - 使用指南

## 环境准备

### 1. 安装 FFmpeg

下载 https://ffmpeg.org/download.html

解压后将 `bin\ffmpeg.exe` 所在目录加入系统环境变量 PATH。

验证：
```bash
ffmpeg -version
```

### 2. 安装 Python 依赖

```bash
pip install moviepy edge-tts pandas openpyxl Pillow
```

### 3. 安装 Git

```bash
git --version
```

---

## 完整操作流程

### Step 1: 准备素材

将原始视频文件（.MOV/.MP4）放入素材文件夹。

如果有镜头索引CSV（产品镜头索引.xlsx），确保放在素材文件夹根目录。

### Step 2: 修改脚本内容

打开 `workflow_60s.py`，修改顶部的 `LINES` 列表：

```python
LINES = [
    "夏天到了，管不住嘴的毛病又犯了，",  # 第1句
    "深夜炸鸡烧烤轮着来，肠胃真的遭不住。",  # 第2句
    # ... 修改为你的文案
]
```

### Step 3: 修改素材路径

```python
BASE_DIR = r"C:\你的\素材\文件夹\路径"
OUT_DIR = r"C:\输出\目录"
```

### Step 4: 运行流水线

```bash
python workflow_60s.py
```

预计耗时：
- TTS生成：30-60秒
- 镜头预处理（20个）：1-2分钟
- 拼接：30秒
- 字幕烧录：2-4分钟
- **总计约5-8分钟**

### Step 5: 检查成品

在 `OUT_DIR` 目录下查找 `成品_素材01_60s.mp4`。

## 进阶：调整TTS参数

在 workflow 脚本中修改：

```python
async def gen_tts():
    communicate = edge_tts.Communicate(
        full_script,
        voice="zh-CN-XiaoxiaoNeural",  # 可选：zh-CN-YunxiNeural（男声）
        rate="+5%",    # 语速：+5%稍快，-5%稍慢
        pitch="-3Hz",   # 音调：+5Hz偏高，-5Hz偏低
    )
    await communicate.save(VO_PATH)
```

## 故障排查

### 报错：subtitles filter 无法解析 original_size
→ 使用 MoviePy 方案（已内置，无需手动处理）

### 报错：MoviePy write_videofile 崩溃
→ 可能是字体路径问题，检查 `C:\Windows\Fonts\arial.ttf` 是否存在

### 成品无声音
→ 确认使用了 `composite.with_audio(tts_audio)` 而非 `video.audio`

### 字幕位置偏上/偏下
→ 修改 `y_pos = video.h - tc.h - 80` 中的 `-80`（底部边距）
